# OMJET Video MerGe v0.1
**Автор:** andrewkwena

---

## Назначение

Программа автоматически склеивает видеозаписи, разбитые на пронумерованные фрагменты, в один цельный файл. Если часть фрагментов отсутствует — на их место вставляется заглушка с чёрным экраном и надписью **«ФРАГМЕНТ УТЕРЯН»**.

---

## Требования

| Компонент | Описание |
|---|---|
| **ffmpeg** | Должен быть установлен и доступен в PATH, либо файлы `ffmpeg.exe` и `ffprobe.exe` должны лежать рядом с программой. Скачать: [gyan.dev/ffmpeg/builds](https://www.gyan.dev/ffmpeg/builds/) → *full build* |
| **Windows 10/11** | 64-разрядная система |
| **GPU** (опционально) | NVIDIA, AMD или Intel — ускоряет создание заглушек. При отсутствии GPU кодирование выполняется программно через CPU |

---

## Формат файлов

Программа ищет файлы строго по шаблону:

```
video_YYYYMMDD_HHMMSS_000.mp4
video_YYYYMMDD_HHMMSS_001.mp4
video_YYYYMMDD_HHMMSS_002.mp4
...
```

- `YYYYMMDD` — дата (например, `20260619`)
- `HHMMSS` — время (например, `142900`)
- `000`, `001`, `002` — порядковый номер фрагмента (три цифры)

Результат сохраняется в ту же папку под именем без номера:
```
video_20260619_142900.mp4
```

---

## Использование

1. Скопировать `OMJET Video MerGe v0.1.exe` в папку с видеофрагментами
2. Запустить двойным кликом
3. Дождаться завершения — программа выведет отчёт и предложит нажать **Enter** для выхода

Программа обрабатывает **все** наборы фрагментов, найденные в папке, за один запуск.

---

## Что происходит при запуске

1. Определяются доступные видеокодеки (NVENC / AMF / QSV / программный)
2. Для каждого набора фрагментов:
   - Считываются параметры исходного видео (кодек, разрешение, FPS, формат пикселей, аудио)
   - Пропущенные фрагменты заменяются заглушками с теми же параметрами
   - Все части склеиваются без перекодирования (stream copy) — качество не теряется
3. Исходные фрагменты **не удаляются**

---

## Выбор кодека (автоматически)

| Приоритет | HEVC (H.265) | H.264 |
|---|---|---|
| 1 | hevc_nvenc (NVIDIA) | h264_nvenc (NVIDIA) |
| 2 | hevc_amf (AMD) | h264_amf (AMD) |
| 3 | hevc_qsv (Intel) | h264_qsv (Intel) |
| 4 | hevc_mf (Windows) | h264_mf (Windows) |
| 5 | libx265 (CPU) | libx264 (CPU) |

Если ни один HEVC-кодек недоступен, заглушки создаются в H.264, а итоговый файл перекодируется — об этом выводится предупреждение.

## Возможные ошибки и решения

| Сообщение | Причина | Решение |
|---|---|---|
| `Не найден 'ffmpeg'` | ffmpeg не установлен или не в PATH | Установить ffmpeg, добавить в PATH или положить `ffmpeg.exe` рядом с программой |
| `Не найдено файлов вида video_...` | В папке нет подходящих файлов | Убедиться, что файлы соответствуют шаблону `video_YYYYMMDD_HHMMSS_NNN.mp4` |
| `[ПРОПУЩЕНО] ... уже существует` | Итоговый файл уже есть в папке | Удалить или переименовать существующий файл и запустить повторно |
| `Не найден ни один видеокодек` | ffmpeg собран без кодеков | Скачать **full build** с сайта gyan.dev |
